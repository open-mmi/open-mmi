# Open MMI vehicle-status registry

> Generated from `canbusd/data/vehicle-statuses.v1.json`. Do not edit this file by hand.

Vehicle profiles translate manufacturer-specific CAN frames into these canonical persistent
status paths. Paths describe human meaning; the registry separately records type, unit,
nullability and lifecycle so downstream code does not need to understand a CAN address.

> **The registry is a continuity checkpoint, not a walled garden.** Raw CAN discovery
> remains open. Reuse an existing human meaning when it fits, or propose a genuinely new
> universal status path in the same pull request as its first maintained vehicle mapping.

See [`vehicle-contribution-workflow.md`](vehicle-contribution-workflow.md) for the open
discovery, reuse and proposal workflow.

## Where hex meets human form

The Seat and Vauxhall CAN layouts may be completely different while the downstream
status remains identical:

```json
{
  "id": "0x470",
  "byte": 1,
  "type": "bitfield",
  "path": "doors",
  "fields": {
    "front_right": "0x01"
  }
}
```

```json
{
  "id": "0x431",
  "byte": 4,
  "type": "bool",
  "path": "doors.front_right",
  "mask": "0x20",
  "true": "0x20",
  "false": "0x00"
}
```

Both publish:

```text
doors.front_right = true
```

The CAN ID, byte, mask and encoding are vehicle-specific. The human meaning is shared.

## Event or status?

- Use an **event** for a momentary request such as `mute_toggle`.
- Use a **status** for persistent state such as `doors.front_right` or
  `parking.distance.rear_left`.
- A discovery label such as `pdc_signal` is not rejected as research. Before it
  becomes maintained, determine whether it is a button event, an active-state flag,
  or a measured parking distance, then reuse or propose the matching canonical concept.

## Lifecycle

- `stable` — supported human-facing contract.
- `experimental` — useful interpretation that still needs confirmation.
- `diagnostic` — raw or reverse-engineering output; not a stable UI contract.
- `deprecated` — retained only for migration.
